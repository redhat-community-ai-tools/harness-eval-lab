# Measuring Impact via Dynamic Workflows

> **Status:** future

## The problem

Impact is the most important evaluation dimension ("does this setup actually help the agent?") and we have no way to measure it. The [impact-dimension](../impact-dimension/) plan explores the problem in depth. This document focuses on one specific solution: using dynamic workflows as the testing machinery.

## The concept

Dynamic workflows provide exactly what impact measurement needs: isolated subagents with controlled context, programmable coordination, and structured output collection. A workflow can spawn Agent A (with setup) and Agent B (without setup), give them the same task, and have Agent J (judge) compare the results.

This is the A/B testing pattern that Anthropic recommends for skill evaluation.

## How it connects to dynamic workflows

The key insight: we don't need to build our own agent orchestration infrastructure. Dynamic workflows already provide:
- **Subagent isolation:** Each agent gets its own context window
- **Worktree isolation:** Agents can work on code without interfering with each other
- **Model routing:** The judge can use a different model than the test agents
- **Structured output:** Schema-based responses from agents, no fragile parsing
- **Resume:** If interrupted, pick up where you left off

Building impact measurement without workflows would mean reimplementing all of this.

## How it would work

```
Workflow: measure-impact
  Phase 1: Generate probe tasks
    -> Agent: generate 3 tasks based on the repo and setup
    
  Phase 2: Run A/B tests (pipeline, not barrier)
    For each probe task:
      -> Agent A: complete the task with full setup loaded
      -> Agent B: complete the task with empty setup
      -> Agent J: blind comparison on 5 dimensions
      
  Phase 3: Aggregate
    -> Compute win rate, dimension deltas
    -> Return impact score (1-5)
```

**Probe tasks** should exercise different aspects:
- A code review task (tests whether review-related skills help)
- A code writing task (tests whether conventions/patterns skills help)
- A debugging task (tests whether debugging skills help)

**The judge** scores both outputs on 5 dimensions (accuracy, specificity, actionability, completeness, response quality) without knowing which had the setup. The winner is determined by total score difference.

## Trade-offs vs other impact measurement approaches

| Aspect | A/B via workflows | Activation rate testing | Differential analysis |
|--------|-------------------|------------------------|----------------------|
| What it measures | Task completion quality | Whether skills trigger | Effect on other dimensions |
| Cost | High (9+ LLM calls) | Low (1 call per skill) | Low (rerun existing analysis) |
| Accuracy | High (direct measurement) | Low (activation != help) | Medium (indirect) |
| Requires | Workflow runtime | Skill trigger mechanism | Existing Layer 1 |
| Determinism | Low | Medium | High |

## How to build it

1. **Workflow script template:** A reusable workflow script in `experiment/measure-impact.js` (or generated by the plugin skill)

2. **Probe task definitions:** Either hardcoded generic tasks or generated by an LLM based on the repo's content. Start with hardcoded, iterate to generated.

3. **Judge rubric:** A structured prompt for the judge agent with the 5 dimension scoring criteria and explicit blindness instructions.

4. **Plugin integration:** The eval-setup SKILL.md adds a final optional step: "If the user wants impact measurement, generate and run the impact workflow."

5. **Score mapping:** Win rate to 1-5 scale: 0-20% wins = 1, 20-40% = 2, 40-60% = 3, 60-80% = 4, 80-100% = 5.

## Open questions

- Should probe tasks be generic (work on any project) or customizable?
- How many runs are needed for statistical stability? (3 probes x 1 run each might not be enough)
- What's the cost ceiling? (9 LLM calls minimum, could be 15-20 with retries)
- Can the judge use a cheaper model (Haiku) or does it need full reasoning (Opus)?
- How to handle the case where the setup helps on some tasks and hurts on others?
